================================================================================
                           1. INTRODUCAO
================================================================================

Este documento descreve os casos de uso do Sistema de Compartilhamento Seguro 
de Arquivos da Petrobras. O sistema permite o envio seguro de documentos 
confidenciais entre colaboradores internos e destinatarios externos, com 
fluxo de aprovacao por supervisores e auditoria completa por administradores.

Objetivo Principal:
- Garantir a transferencia segura de arquivos confidenciais
- Manter rastreabilidade completa de todas as operacoes
- Implementar fluxo de aprovacao hierarquico
- Proteger informacoes sensiveis da organizacao

================================================================================
                       2. VISAO GERAL DO SISTEMA
================================================================================

O sistema possui quatro perfis de usuarios com diferentes niveis de acesso:

+------------------+--------------------------------------------------+
|     PERFIL       |              DESCRICAO                           |
+------------------+--------------------------------------------------+
| Usuario Interno  | Colaborador Petrobras que envia arquivos         |
| Usuario Externo  | Destinatario externo que recebe arquivos         |
| Supervisor       | Aprova/rejeita compartilhamentos e envia arquivos|
| Admin Global     | Administra o sistema completo                    |
+------------------+--------------------------------------------------+

================================================================================
                        3. ATORES DO SISTEMA
================================================================================

ATOR: Usuario Interno (UI)
--------------------------
Descricao: Colaborador da Petrobras autenticado via Microsoft Entra ID
Responsabilidades:
  - Fazer upload de arquivos para compartilhamento
  - Definir destinatario externo
  - Acompanhar status dos envios
  - Visualizar historico de compartilhamentos

ATOR: Usuario Externo (UE)
--------------------------
Descricao: Pessoa externa a Petrobras que recebe arquivos compartilhados
Responsabilidades:
  - Receber codigo de verificacao por e-mail
  - Acessar sistema com autenticacao temporaria
  - Fazer download dos arquivos aprovados
  - Confirmar recebimento

ATOR: Supervisor (SUP)
----------------------
Descricao: Gestor responsavel por aprovar compartilhamentos de sua equipe
Responsabilidades:
  - Aprovar ou rejeitar solicitacoes de compartilhamento
  - Visualizar detalhes dos arquivos
  - Acompanhar metricas da equipe
  - Realizar compartilhamentos com aprovacao automatica

ATOR: Admin Global (ADM)
------------------------
Descricao: Administrador com acesso total ao sistema
Responsabilidades:
  - Gerenciar usuarios
  - Visualizar todos os compartilhamentos
  - Acessar logs de auditoria
  - Rastrear atividades de usuarios
  - Monitorar metricas do sistema

+----------------------------------------------------------------------------+
|  UC-UI-001: LOGIN NO SISTEMA                                               |
+----------------------------------------------------------------------------+
| Ator Principal: Usuario Interno                                            |
| Pre-condicoes: Usuario possui conta corporativa Microsoft                  |
| Pos-condicoes: Usuario autenticado com acesso ao dashboard                 |
+----------------------------------------------------------------------------+
| FLUXO PRINCIPAL:                                                           |
| 1. Usuario acessa a pagina inicial do sistema                              |
| 2. Sistema exibe tela de login com botao "Login corporativo"               |
| 3. Usuario clica no botao "Login corporativo"                              |
| 4. Sistema exibe loading com logo Petrobras                                |
| 5. Sistema redireciona para autenticacao Microsoft Entra ID                |
| 6. Usuario insere credenciais corporativas                                 |
| 7. Microsoft valida credenciais e retorna token                            |
| 8. Sistema valida token e cria sessao                                      |
| 9. Sistema identifica tipo de usuario (interno/supervisor)                 |
| 10. Sistema redireciona para dashboard apropriado                          |
+----------------------------------------------------------------------------+
| FLUXOS ALTERNATIVOS:                                                       |
| 3a. Usuario clica em "Acessar documentacao"                                |
|     - Sistema exibe pagina de documentacao                                 |
| 7a. Credenciais invalidas                                                  |
|     - Microsoft exibe mensagem de erro                                     |
|     - Usuario pode tentar novamente                                        |

+----------------------------------------------------------------------------+
|  UC-UI-002: FAZER UPLOAD DE ARQUIVOS                                       |
+----------------------------------------------------------------------------+

+----------------------------------------------------------------------------+
| FLUXO PRINCIPAL:                                                           |
| 1. Usuario acessa a tela de "Transferencia Segura de Arquivos"             |

|    - Campo de e-mail do destinatario externo                               |

|    - Informacao do supervisor aprovador                                    |
| 3. Usuario informa e-mail do destinatario externo                          |
| 4. Usuario arrasta ou seleciona arquivos para upload                       |
| 5. Sistema valida extensoes dos arquivos (bloqueia .exe, .bat, etc.)       |
| 6. Usuario seleciona tempo de disponibilidade                              |
| 7. Usuario preenche descricao do envio                                     |
| 8. Usuario clica em "Enviar para Aprovacao"                                |
| 9. Sistema exibe barra de progresso do upload                              |
| 10. Sistema envia arquivos para armazenamento seguro (S3)                  |
| 11. Sistema cria registro de compartilhamento com status "pendente"        |
| 12. Sistema notifica supervisor por e-mail                                 |
| 13. Sistema exibe modal de sucesso com detalhes do envio                   |

| FLUXOS ALTERNATIVOS:                                                       |
| 5a. Arquivo com extensao bloqueada detectado                               |
|     - Sistema exibe notificacao de erro                                    |
|     - Lista arquivos bloqueados e extensoes proibidas                      |
|     - Usuario deve remover arquivos invalidos                              |

|     - Sistema exibe mensagem de erro                                       |
|     - Nenhum registro e criado                                             |
|     - Usuario pode tentar novamente                                        |

| REGRAS DE NEGOCIO:                                                         |
| - Extensoes bloqueadas: .exe, .dll, .bat, .cmd, .com, .msi, .scr,          |
|   .vbs, .ps1, .sh                                                          |
| - Tempo maximo de disponibilidade: 168 horas (7 dias)                      |
| - Descricao e obrigatoria                                                  |
| - E-mail do destinatario deve ser valido                                   |

================================================================================
                  9. FLUXOS ALTERNATIVOS E EXCECOES
================================================================================

EXCECAO: SESSAO EXPIRADA
------------------------
Gatilho: Token de autenticacao expirou durante uso
Fluxo:
1. Sistema detecta token expirado
2. Sistema redireciona para pagina de login
3. Sistema exibe mensagem "Sessao expirada"
4. Usuario deve fazer login novamente

EXCECAO: ERRO DE CONEXAO
------------------------
Gatilho: Falha de rede durante operacao
Fluxo:
1. Sistema detecta falha de conexao
2. Sistema exibe notificacao de erro
3. Usuario pode tentar novamente

EXCECAO: ARQUIVO MUITO GRANDE
-----------------------------
Gatilho: Usuario tenta enviar arquivo maior que o limite
Fluxo:
1. Sistema detecta tamanho do arquivo
2. Sistema exibe mensagem de erro com limite permitido
3. Usuario deve selecionar arquivo menor

EXCECAO: COMPARTILHAMENTO EXPIRADO
----------------------------------
Gatilho: Usuario externo tenta acessar compartilhamento expirado
Fluxo:
1. Sistema verifica data de expiracao
2. Sistema exibe mensagem "Compartilhamento expirado"
3. Usuario externo deve solicitar novo compartilhamento

================================================================================
                    10. REQUISITOS NAO FUNCIONAIS
================================================================================

SEGURANCA:
- Autenticacao via Microsoft Entra ID (SSO)
- Tokens JWT com expiracao
- Criptografia em transito (HTTPS)
- Arquivos armazenados em S3 com criptografia
- Validacao de extensoes de arquivos
- Registro de todas as acoes em log de auditoria
- Codigo de verificacao temporario para usuarios externos

DESEMPENHO:
- Tempo de resposta < 3 segundos
- Upload com barra de progresso
- Paginacao de listas grandes
- Cache de dados quando apropriado

USABILIDADE:
- Interface responsiva (mobile e desktop)
- Feedback visual para todas as acoes
- Mensagens de erro claras e acionaveis
- Documentacao integrada ao sistema

DISPONIBILIDADE:
- Sistema disponivel 24/7
- Backup regular dos dados
- Notificacoes de manutencao programada
