Permissão IAM necessária — Secrets Manager

Aplicação CSA A12022 · Ambiente DSV · ECS Task Role

1. Contexto

A aplicação CSA A12022 (backend FastAPI rodando no ECS) precisa conectar-se ao banco de dados RDS Aurora PostgreSQL. As credenciais de acesso ao banco estão armazenadas no AWS Secrets Manager, que é a prática recomendada pela AWS para não expor senhas como variáveis de ambiente na Task Definition.

Para que a aplicação leia essas credenciais em tempo de execução, a IAM Task Role do ECS precisa ter permissão secretsmanager:GetSecretValue nas secrets do banco. Atualmente essa permissão está ausente, causando o erro abaixo.

2. Erro Atual

AccessDeniedException retornado pela aplicação em produção (ECS DSV):

User: arn:aws:sts::119392112451:assumed-role/AppSpace-ecs-task-a12022--ecstaska12022dsvbackendta-N24Vt2E5AAq4/96a4cb44dcf147d9b72f0fa6dccb6cec is not authorized to perform: secretsmanager:GetSecretValue on resource: db_aplicacao_psql_dsv_a12022_secret because no identity-based policy allows the secretsmanager:GetSecretValue action

Adicionar a permissão secretsmanager:GetSecretValue restrita às duas secrets de banco da aplicação na IAM Task Role do ECS:

Permissão Situação atual Situação necessária
ssmmessages:* ✔ Presente ✔ Manter
cloudwatch:PutMetricData ✔ Presente ✔ Manter
logs:CreateLogStream / PutLogEvents ✔ Presente ✔ Manter
secretsmanager:GetSecretValue ✘ Ausente ✔ Adicionar

3. O que Precisa Ser Feito

IAM Role (Task Role ECS) AppSpace-ecs-task-a12022--ecstaska12022dsvbackendta-N24Vt2E5AAq4
Conta AWS119392112451
Regiãosa-east-1 (São Paulo)
Secret 1db_aplicacao_psql_dsv_a12022_secret — credencial de aplicação (DML)
Secret 2db_owner_psql_dsv_a12022_secret — credencial owner (DDL / Alembic)

4. Policy JSON (Inline Policy)

{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "AllowReadDbSecrets",
      "Effect": "Allow",
      "Action": "secretsmanager:GetSecretValue",
      "Resource": [
        "arn:aws:secretsmanager:sa-east-1:119392112451:secret:db_aplicacao_psql_dsv_a12022_secret*",
        "arn:aws:secretsmanager:sa-east-1:119392112451:secret:db_owner_psql_dsv_a12022_secret*"
      ]
    }
  ]
}
O * no final dos ARNs é necessário porque o Secrets Manager adiciona automaticamente um sufixo aleatório de 6 caracteres ao ARN de cada secret (ex.: ...secret-Ab1Cd2).

5. Passos para Execução (Console AWS)

1

Localizar a Role

IAM Console → Roles → buscar por:
AppSpace-ecs-task-a12022--ecstaska12022dsvbackendta-N24Vt2E5AAq4

2

Adicionar permissão

Clique em Add permissions → Create inline policy
Selecione a aba JSON → cole a policy da seção 4 → clique em Review policy

3

Nomear e salvar

Nome sugerido para a policy: AllowReadDbSecrets-A12022-DSV
Clique em Create policy.

4

Validar (opcional)

Após o deploy ou restart da task ECS, chamar o endpoint de diagnóstico:
GET /api/v1/diagnostico/db-secrets
Resultado esperado: "secret_manager": "ok" para ambas as secrets.

6. Alternativa via AWS CLI

aws iam put-role-policy \
  --role-name "AppSpace-ecs-task-a12022--ecstaska12022dsvbackendta-N24Vt2E5AAq4" \
  --policy-name "AllowReadDbSecrets-A12022-DSV" \
  --policy-document '{
    "Version": "2012-10-17",
    "Statement": [{
      "Sid": "AllowReadDbSecrets",
      "Effect": "Allow",
      "Action": "secretsmanager:GetSecretValue",
      "Resource": [
        "arn:aws:secretsmanager:sa-east-1:119392112451:secret:db_aplicacao_psql_dsv_a12022_secret*",
        "arn:aws:secretsmanager:sa-east-1:119392112451:secret:db_owner_psql_dsv_a12022_secret*"
      ]
    }]
  }'

7. Como Validar

Após aplicar a policy (sem necessidade de redeploy da task), chamar o endpoint de diagnóstico da aplicação:

GET /api/v1/diagnostico/db-secrets

Resultado esperado:

{
  "resumo": {
    "db_aplicacao_psql_dsv_a12022_secret": "[Usuário aplicação (DML)] CONECTADO ...",
    "db_owner_psql_dsv_a12022_secret":     "[Usuário owner (DDL/migrations)] CONECTADO ..."
  }
}
Escopo mínimo: A permissão solicitada é restrita exclusivamente às duas secrets de banco de dados do ambiente DSV. Nenhuma outra secret ou recurso AWS é incluído nesta política.
